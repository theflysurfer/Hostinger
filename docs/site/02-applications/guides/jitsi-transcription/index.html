    A. ImageHash - Perceptual Hashing ‚≠ê Recommand√© pour slides¶

    Concept : G√©n√©rer un "hash perceptuel" de chaque frame. Deux images similaires auront des hashes similaires.

    GitHub : https://github.com/JohannesBuchner/imagehash Stars : 3.2k | Derni√®re release : 1er f√©vrier 2025

    Pourquoi c'est parfait pour les slides ?¶

    ‚úÖ Hash perceptuel : 2 images similaires = hashes similaires ‚úÖ Distance de Hamming : Mesure num√©rique de la diff√©rence (0 = identiques, 64 = tr√®s diff√©rents) ‚úÖ Insensible aux variations : Compression vid√©o, petits mouvements de curseur ignor√©s ‚úÖ Ultra rapide : Hash en <1ms par image

    Algorithmes disponibles¶
    Algorithme Usage Vitesse Pr√©cision
    average_hash Simple, slides statiques ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê ‚≠ê‚≠ê‚≠ê
    phash (perceptual) ‚≠ê Meilleur pour slides ‚≠ê‚≠ê‚≠ê‚≠ê ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê
    dhash (difference) D√©tection bordures ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê ‚≠ê‚≠ê‚≠ê‚≠ê
    whash (wavelet) Tr√®s pr√©cis ‚≠ê‚≠ê‚≠ê ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê

    import imagehash
from PIL import Image
from pathlib import Path

def detect_slide_changes(frames: list[str], threshold: int = 5) -> list[int]:
    """
    D√©tecte les changements de slides en comparant les hashes perceptuels.

    Args:
        frames: Liste de chemins vers les images (frames extraits)
        threshold: Seuil de distance Hamming
            - 0-5  : Images quasi identiques (m√™me slide)
            - 6-10 : Petites variations (curseur, animation)
            - 10+  : Changement significatif (nouvelle slide)

    Returns:
        Liste des indices de frames o√π un changement de slide est d√©tect√©

    Example:
        >>> frames = ["frame_0001.jpg", "frame_0002.jpg", ...]
        >>> changes = detect_slide_changes(frames, threshold=8)
        >>> print(changes)
        [45, 123, 289, 456]  # Nouvelles slides aux frames 45, 123, 289, 456
    """

    changes = []
    prev_hash = None

    for i, frame_path in enumerate(frames):
        img = Image.open(frame_path)

        # G√©n√©rer hash perceptuel (16x16 = 256 bits)
        current_hash = imagehash.phash(img, hash_size=16)

        if prev_hash is not None:
            # Distance de Hamming = nombre de bits diff√©rents
            distance = current_hash - prev_hash

            if distance > threshold:
                changes.append(i)
                print(f"üìä Changement de slide d√©tect√© : frame {i} (distance: {distance})")

        prev_hash = current_hash

    return changes

    Approche 2 : Analyse heuristique des frames (custom)¶

    Principe : Distinguer webcam vs screenshare par analyse d'image OpenCV

    Caract√©ristiques visuelles¶
    Webcam Screenshare
    Formes floues, peau Contours nets, texte

    Code : D√©tection screenshare par heuristiques¶
    import cv2
import numpy as np
from paddleocr import PaddleOCR

# Initialiser OCR (r√©utilis√© pour chaque frame)
ocr = PaddleOCR(use_angle_cls=False, lang='en', use_gpu=False, show_log=False)

def is_screenshare_frame(frame: np.ndarray) -> tuple[bool, float]:
    """
    D√©termine si une frame est un screenshare ou une webcam.

    Args:
        frame: Image numpy array (BGR)

    Returns:
        (is_screenshare, confidence_score)

    Example:
        >>> cap = cv2.VideoCapture('meeting.mp4')
        >>> ret, frame = cap.read()
        >>> is_screen, score = is_screenshare_frame(frame)
        >>> print(f"Screenshare: {is_screen} (score: {score:.2%})")
        Screenshare: True (score: 0.34)
    """

    frame_area = frame.shape[0] * frame.shape[1]

    # ==========================
    # 1. D√©tection de TEXTE
    # ==========================
    # Beaucoup de texte = probablement screenshare

    result = ocr.ocr(frame, cls=False)

    text_area = 0
    if result and result[0]:
        for detection in result[0]:
            bbox = detection[0]
            # Calculer aire de la bounding box
            width = bbox[1][0] - bbox[0][0]
            height = bbox[2][1] - bbox[1][1]
            text_area += width * height

    text_ratio = text_area / frame_area

    # ==========================
    # 2. D√©tection de CONTOURS
    # ==========================
    # Slides = beaucoup de contours nets (bordures, s√©parateurs)

    gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
    edges = cv2.Canny(gray, 50, 150)
    edge_ratio = np.count_nonzero(edges) / frame_area

    # ==========================
    # 3. Analyse COULEUR
    # ==========================
    # UI = beaucoup de blanc/gris (backgrounds PowerPoint, Google Docs)

    hsv = cv2.cvtColor(frame, cv2.COLOR_BGR2HSV)

    # Masque blanc (Saturation faible, Value √©lev√©e)
    white_mask = cv2.inRange(hsv, (0, 0, 200), (180, 30, 255))
    white_ratio = np.count_nonzero(white_mask) / frame_area

    # ==========================
    # 4. SCORE COMPOSITE
    # ==========================

    score = (
        text_ratio * 0.5 +      # 50% poids texte (crit√®re principal)
        edge_ratio * 0.3 +      # 30% poids contours
        white_ratio * 0.2       # 20% poids blanc
    )

    # Seuil : ajuster selon tests
    # - Webcam typique : score 0.05-0.10
    # - Screenshare typique : score 0.20-0.50
    is_screenshare = score > 0.15

    return is_screenshare, score

# ==========================
